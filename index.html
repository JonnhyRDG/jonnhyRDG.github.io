GIT

Frequently used commands and basic process:

  • git init ← start a local repo wherever you execute this. Will create .git folder.
  • git add . ← this will add all your files to the stage state, if files have been modified from the last commit. “.” means all files in the repo. When you specify each file, you can concatenate strings of files.
  • git commit -m “message” ← this creates a checkpoint at the time of saving.
  • git checkout -b “branch name” ← Creates new branch (that’s what the -b is for)
  • git checkout “branch name” ← Switch to branch.
  • git merge master ← Merge master into the active branch
  • git remote add origins <repo url> ← mapping the local repo to the remote one created in github
  • git push -u origin master ← this will push your new stuff to the remote repo.
  • git pull origin master ← getting the changes from the remote desktop (to the current active branch)
  • git push --set-upstream origin qtree ← set an upstream branch

Sample:

        File change

  • git add “file” (into stage)

        

        Save changes

  • git commit -m “newchange”

        Push change to remote:

  • git push -u origin qtree

Do the merge in github

  • In github, pull the merge request and compare.

Now update master(local) with origin

  • git pull origin Master
